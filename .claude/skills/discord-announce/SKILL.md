# Discord 공지 초안 작성

## 목적

내부 변경사항이나 클럽 공지를 Discord 채널에 맞는 포맷으로 초안을 작성하고 `01_ops/posts/`에 저장한다. 직접 발행은 사람이 한다 (반자동화).

## 입력

| 인자 | 필수 | 설명 |
|------|------|------|
| `채널유형` | 선택 | `ops` / `announce` / `showcase` (없으면 `ops`) |
| `주제` | 선택 | 공지 주제 키워드 (없으면 사용자에게 물어봄) |

인자 없이 호출하면 채널 유형과 주제를 사용자에게 확인한다.

---

## 채널 유형별 특성

| 유형 | 대상 채널 | 대상 독자 | 톤 | 길이 |
|------|----------|----------|---|------|
| `ops` | `#ops-decision-log` / `#ops-general` | 운영팀 | 실무적, 간결 | 중간 (400–700자) |
| `announce` | `#announcements` | 전체 멤버 | 공식적, 친근 | 짧음 (200–400자) |
| `showcase` | `#showcase` | 전체 멤버 | 축하, 공유 | 자유 |

---

## 절차

### 1. 컨텍스트 파악

아래 파일을 읽어 현재 상태를 파악한다:

```
_system/agents/memory/active-context.md   ← 클럽 현재 상태
_system/agents/memory/change-log.md       ← 최근 변경사항
```

주제가 주어지지 않으면 `change-log.md`의 가장 최근 날짜 항목에서 공지할 내용을 파악한다.

### 2. 소스 파일 확인 (있으면)

주제와 관련된 기존 문서가 있으면 읽어서 내용을 보강한다.

### 3. 초안 작성 — 채널 유형별 포맷

#### `ops` — 운영팀 내부 (`#ops-decision-log`)

```
**🔧 [제목]** | YYYY-MM-DD

[1-2문장 배경 설명]

**① [항목명]**
[간결한 설명 1-2줄]

**② [항목명]**
[간결한 설명 1-2줄]

(항목 반복)

---

[마무리 — 다음 행동 또는 피드백 요청 1줄]
```

#### `announce` — 전체 멤버 공지 (`#announcements`)

```
**📢 [제목]**

[핵심 내용 1-3줄. 링크나 날짜 포함]

자세한 내용: [링크 또는 채널]
```

#### `showcase` — 결과 공유 (`#showcase`)

```
**✨ [제목]**

[무엇을 만들었는지 1-2줄]

[이미지 또는 스크린샷 첨부 안내 (있으면)]

[관련 링크 또는 참고 채널]
```

### 4. 이미지 가이드 생성

공지 내용을 보완할 이미지가 필요한지 판단하고, 이미지 유형을 결정한다.

#### 이미지 유형 판단

| 상황 | 유형 | 방법 |
|------|------|------|
| 개념·계층·흐름 설명 | **다이어그램** | 에이전트가 Mermaid 코드 생성 → 사용자가 렌더링 후 캡처 |
| 폴더 구조·파일 목록 | **다이어그램** | Mermaid `graph` 또는 ASCII tree 생성 |
| 실제 UI 화면 보여주기 | **시스템 캡처** | 에이전트가 캡처 위치·방법 안내 |
| 노트 내용 시각화 | **시스템 캡처** | 해당 노트 열고 전체 화면 캡처 |
| 변경 전후 비교 | **시스템 캡처 2장** | 각각 캡처 방법 안내 |

#### 다이어그램 생성 규칙

- Mermaid `graph TD` 또는 `flowchart TD` 사용 (Discord에서 직접 렌더 안 됨 — 이미지로 변환 필요)
- 복잡한 다이어그램은 Excalidraw용 JSON 명세 작성 가능
- 다이어그램은 초안 파일의 `## 다이어그램 소스` 섹션에 포함

#### 이미지 파일명 규칙

```
YYMMDD-{설명-슬러그}.png
```

예시: `260324-moc-map-index-계층.png`, `260324-maps-폴더트리.png`

저장 위치: `01_ops/posts/assets/`

#### 이미지 크기 표준

| 용도 | 크기 | 예시 상황 |
|------|------|----------|
| 메인 다이어그램 | `\|700` | 계층 구조, 개념 흐름도 |
| 시스템 캡처 | `\|600` | 노트 전체, 폴더 트리 |
| 보조 참고 | `\|400` | 부분 캡처, 설정 화면 |

#### 이미지 모듈 (발행문 안에서 반복 사용)

각 섹션 텍스트 코드블록 바로 다음에 이미지 + 캡션을 배치한다.
이미지와 캡션은 코드블록 **밖**에 위치한다.

```markdown
**[① 섹션명]** 복사 후 붙여넣기
\```
**① 섹션명**
설명 텍스트
\```

![[YYMMDD-파일명.png|600]]
*캡션 — 이 이미지가 무엇을 보여주는지 한 줄*
```

**규칙:**
- 이미지는 해당 섹션 코드블록 바로 아래 — 텍스트와 이미지를 한 번에 발행
- 캡션은 이탤릭 한 줄 (`*텍스트*`)
- 이미지 없는 섹션은 코드블록만, 이미지 모듈 생략
- `.png.png` 이중 확장자 금지 — 파일명에 이미 `.png` 포함 시 `|크기`만 추가

#### 이미지 가이드 테이블 포맷

```markdown
## 이미지 가이드

저장 위치: `01_ops/posts/assets/`

| # | 파일명 | 유형 | 소스 / 위치 | 방법 |
|---|--------|------|------------|------|
| 1 | `YYMMDD-설명.png` | 다이어그램 | 아래 Mermaid 소스 | 렌더 후 스크린샷 |
| 2 | `YYMMDD-설명.png` | 시스템 캡처 | [Obsidian 경로] | [열기 방법 + 상태] |
```

#### 발행문 안에 이미지 위치 표시

```
**① [항목명]**
[설명 1-2줄]
```
[이미지 N — 다이어그램/캡처: 설명]
```
```

이미지가 필요 없는 공지(단순 일정·링크 전달 등)는 이 섹션을 생략한다.

### 5. 파일 저장

파일명: `YYMMDD-{채널유형}-{주제슬러그}.md`
저장 위치: `01_ops/posts/`

```yaml
---
title: [제목]
kind: post
area: ops
status: draft
created: YYYY-MM-DD
target_channel: "[채널명]"
tags:
  - ops/post
---
```

파일 구조:
1. 이미지 캡처 가이드 (이미지가 있을 경우)
2. Discord 발행문 (코드 블록으로 감싸 복사하기 쉽게, 이미지 위치 표시 포함)
3. 발행 체크리스트
   - [ ] 이미지 N 캡처: [설명] (이미지 항목별)
   - [ ] 발행문 + 이미지 순서대로 [채널]에 붙여넣기
   - [ ] 발행 후 status → `live`

### 6. 출력

- 저장된 파일 경로 안내
- 발행문 전체를 대화창에 출력 (바로 복사할 수 있게)
- 이미지 캡처 목록 간략 안내
- 대상 채널 명시

---

## 발행 워크플로우

```
/discord-announce ops "시스템 업데이트"
   ↓
에이전트: change-log 읽기 → 초안 + 이미지 가이드 작성 → 01_ops/posts/ 저장
   ↓
사용자: 이미지 캡처 (가이드 참고) → 내용 확인
   ↓
Discord: 발행문 붙여넣기 → 이미지 첨부 (순서대로)
   ↓
발행 완료 후 status → live
```

---

## 규칙

- Discord markdown 사용: `**굵게**`, `` `코드` ``, `> 인용`
- 이모지는 섹션 헤더에만 1개 (과도한 사용 금지)
- 한국어 기본, 기술 용어·코드는 영어 유지
- 로컬 문서 링크는 Discord에서 열리지 않으므로 요약 포함
- `01_ops/posts/` — 발행 전 초안. 발행 완료 후 `05_library/published/`로 이동은 선택사항
