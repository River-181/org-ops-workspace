# session-transcript

[녹음 앱] 원문 녹음본을 두 가지 산출물로 변환한다:

- **정제 스크립트** (`세션스크립트.md`): 오염만 제거, 원문 보존 — 요약·재구성 금지
- **세션 정리** (`세션정리.md`): 섹션별 핵심 개념·흐름을 정리한 요약본

두 파일 모두 `member-share-space/{세션폴더}/`와 `materials/`에 저장한다.

---

## 입력

- **세션 ID**: `NN` 또는 `S{NN}` (예: `01`, `S01`)
- **녹음본 경로** (선택): 기본값 → `02_sessions/S{NN}-*/materials/*세션녹음본.md`

---

## 사전 준비: 화자 매핑

작업 시작 전 `S{NN}-{슬러그}-plan.md` 또는 `S{NN}-{슬러그}-record.md`에서
발표자 정보를 읽어 [녹음 앱] 화자 번호와 실제 이름을 매핑한다.

```
참석자 1 → {이름} (1부 발표자)
참석자 2 → {이름} (진행 담당)
참석자 3 → {이름} (2부 발표자)
```

매핑이 불분명하면 발화 내용·맥락으로 추론하거나 사용자에게 확인한다.

---

## Phase 1: 정제 스크립트 — `YYMMDD-S{NN}-세션스크립트.md`

**핵심 원칙**: 요약하지 않는다. 강의 필기처럼 내용을 압축하거나 재구성하지 않는다.
원문에서 오염(노이즈)만 걷어내는 작업이다.

### 제거 대상

| 항목 | 예시 |
|------|------|
| 타임스탬프 | `00:18`, `1:23:45` |
| 화자 번호 라인 | `참석자 1 00:18` (이름으로 대체) |
| 추임새 | `음`, `어`, `아`, `그`, `뭐`, `네네네`, `맞죠`, `그니까`, `저기` |
| STT 오류 | `해걸` → `해결`, 문맥상 맞지 않는 단어 |
| 불완전 조각 | 의미 없는 한두 단어만 있는 줄 |
| 비교육적 잡담 | 입장 소동, 기기 설정 대화, 장소 관련 잡담 |
| [녹음 앱] 헤더 | 공유 링크, 비밀번호, 날짜·시간 메타데이터 블록 |

### 보존 대상

- 실제 강의·발표 내용 전문
- 질의응답 (Q&A)
- 사례, 데모, 비유 설명
- 발표자의 원문 어체 (존댓말·반말 그대로 유지)

### STT 오류 처리

- **확실한 오류**: 교정 (예: `해걸` → `해결`)
- **불확실한 오류**: `[?]` 표기 후 원문 유지 (예: `인지[?]`)
- 녹음본 원문(`*세션녹음본.md`)은 절대 수정하지 않는다

### 출력 형식

```yaml
---
title: "S{NN} 세션 스크립트 — {제목}"
kind: session-script
area: sessions
session_id: S{NN}
status: live
created: YYYY-MM-DD
---
```

본문 구조:
```
## 1부 — {1부 제목}

**{이름}**: 그러면 시작하겠습니다. 오늘 주제는...

**{이름}**: Karpathy가 말하기를...

---

## 휴식

---

## 2부 — {2부 제목}

**{이름}**: 안녕하세요. 2부를 시작하겠습니다...
```

- 화자: `**이름**: 발화 내용` 형식
- 섹션 구분: plan.md의 아젠다 기준 `##` 헤더
- 섹션 간 구분선 `---`
- 긴 발화는 자연스러운 의미 단위로 단락 분리

### 저장

1. `02_sessions/member-share-space/{공유폴더}/YYMMDD-S{NN}-세션스크립트.md`
2. `02_sessions/S{NN}-{슬러그}/materials/YYMMDD-S{NN}-세션스크립트.md`

---

## Phase 2: 세션 정리 — `YYMMDD-S{NN}-세션정리.md`

Phase 1 정제 스크립트를 바탕으로 교육 내용 중심의 정리 문서를 작성한다.

### 출력 형식

```yaml
---
title: "S{NN} 세션 정리 — {제목}"
kind: session-summary
area: sessions
session_id: S{NN}
status: live
created: YYYY-MM-DD
---
```

본문 구조:

```markdown
## 1부 — {1부 제목}

### 핵심 개념

- **{개념1}**: {한 줄 설명}
- **{개념2}**: {한 줄 설명}

### 주요 발언

> "{짧은 인상적 인용}" — {발화자}

### 사례

- **{사례 이름}**: {설명}

---

## 2부 — {2부 제목}

### 핵심 개념

...

### 실습 흐름

1. {실습 1단계}
2. {실습 2단계}

**결과물**: {실습 완료 후 생성된 것}

---

## Q&A

**Q**: {질문 내용}
**A**: {답변 요약}
```

### 저장 + record.md 연결

1. `02_sessions/member-share-space/{공유폴더}/YYMMDD-S{NN}-세션정리.md`
2. `02_sessions/S{NN}-{슬러그}/materials/YYMMDD-S{NN}-세션정리.md`
3. `S{NN}-{슬러그}-record.md`의 `materials:` 배열에 두 파일 링크 추가

---

## 긴 녹음본 처리 (1000줄 이상)

섹션 단위(`## 1부`, `## 2부`)로 나누어 처리한다:
1. 원문 전체를 먼저 스캔하여 섹션 경계 파악
2. 섹션별로 정제 작업 수행
3. 합쳐서 최종 파일 저장

---

## 완료 보고

```
✅ session-transcript 완료 — S{NN}

Phase 1 — 정제 스크립트
  저장: member-share-space/.../YYMMDD-S{NN}-세션스크립트.md
  저장: materials/YYMMDD-S{NN}-세션스크립트.md
  원문 {N}줄 → 정제본 {M}줄

Phase 2 — 세션 정리
  저장: member-share-space/.../YYMMDD-S{NN}-세션정리.md
  저장: materials/YYMMDD-S{NN}-세션정리.md

record.md materials 배열 갱신 완료
```

---

## 주의사항

- **Phase 1은 요약 금지**: 문장 압축, 내용 합치기, 재구성 모두 금지
- 녹음본 원문(`*세션녹음본.md`)은 절대 수정하지 않는다
- 공유 폴더가 없으면 먼저 `/session-package {세션ID}`를 실행한다

---

## 연관 스킬

| 스킬 | 관계 |
|------|------|
| `/session-package {세션ID}` | 공유 폴더 생성 — transcript 전에 실행 권장 |
| `/session-close {세션ID}` | 세션 종료 후 허브 파일·MOC 갱신 |
| `/session-ops record {세션폴더}` | record.md 작성 |
