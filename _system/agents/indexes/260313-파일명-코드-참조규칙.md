# 파일명 규칙 인덱스

이 문서는 active `[볼트명]`에서 새 파일을 만들 때 따르는 **현재 규칙**을 요약한다.  
legacy 코드형 파일명은 `05_library/seasons/2025-pre/legacy-hold/`와 기존 보존 문서에서만 유지한다.

## 1. 기본 원칙

- 기본 파일명은 `YYMMDD-제목.확장자`
- 파일명 상태값(`_live`, `_draft`, `_arch`)은 쓰지 않는다
- 상태는 파일명이 아니라 frontmatter `status:`에 둔다
- 파일명 공백은 쓰지 않고 하이픈(`-`)을 사용한다
- 최신 운영본은 가장 단순한 이름을 유지하고, 필요한 경우에만 `-v02`를 붙인다

## 2. 현재 표준 패턴

### 일반 문서

```text
YYMMDD-제목.md
```

예시:

```text
260314-클럽타임라인.md
260320-시즌1모집공고.md
250708-동아리설립방법론.md
```

### 템플릿

```text
tpl-제목.md
```

예시:

```text
tpl-회의록.md
tpl-세션기획.md
```

### 세션 폴더

```text
S{NN}-제목/
```

예시:

```text
S01-내우주의시작/
S02-노션으로시작하는세컨드브레인/
```

### 프로젝트 폴더

```text
PRJ-이름/
```

예시:

```text
PRJ-워크스페이스개편/
PRJ-노션DB세팅/
```

## 3. 예외

- machine-generated 산출물은 기존 접두사 규칙을 유지할 수 있다.
  - 예: `TMP-WEB-260314-01_T2_[볼트명]_워크스페이스맵_v01.html`
- legacy에서 막 가져온 문서는 history/reference/archive 성격이면 기존 frontmatter를 유지할 수 있다.
- 외부 자산, 이미지, export는 의미를 보존하기 위해 `YYMMDD-주제-변형.ext` 형태를 허용한다.

## 4. 금지 규칙

- 새 active 문서에 `AREA-UNIT-...` 코드형 이름 부여
- 새 active 문서에 `_live`, `_draft`, `_arch` 접미사 부여
- 파일명에 공백 사용
- 같은 폴더에 같은 제목의 중복 최신본 생성

## 5. 버전 관리

- 기본값: 버전 표기 없음
- 수정 이력이 중요한 경우만 `-v02`, `-v03` 사용
- 이전 버전은 `history/`, `legacy-hold/`, 또는 git으로 관리

## 6. 적용 순서

1. 먼저 대상 폴더의 기존 파일명 관행을 확인한다.
2. active 운영 문서면 `YYMMDD-제목` 규칙을 적용한다.
3. 세션/프로젝트/템플릿이면 전용 접두사(`S`, `PRJ`, `tpl`)를 쓴다.
4. legacy 보존본이면 이름을 억지로 정규화하지 말고 archive 성격에 맞게 둔다.
5. 애매하면 `06_inbox/triage/`에 두고 `_system/agents/indexes/260313-폴더-라우팅-인덱스.md`를 먼저 본다.
