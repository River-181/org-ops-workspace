# 폴더 라우팅 인덱스

## 1. 최상위 라우팅

| 대상 | 위치 |
|------|------|
| 워크스페이스 규칙, 에이전트 운영, 플랫폼 연동 | `_system/` |
| 운영 전략, 회의, 모집, 재정, 멤버, 운영 템플릿 | `01_ops/` |
| 정기/비정기 세션 기획, 자료, 기록 | `02_sessions/` |
| 장기 실행 단위 프로젝트 | `03_projects/` |
| 브랜드, 디자인 소스, 자산, export | `04_studio/` |
| 리서치, 참고 자료, 발행본, 시즌 아카이브 | `05_library/` |
| 아직 분류되지 않은 파일 | `06_inbox/` |

## 2. 하위 폴더 빠른 매핑

| 문서/파일 성격 | 목적지 |
|---------------|--------|
| 정체성, 헌법 초안, foundation 역사본 | `_system/foundation/` |
| 에이전트 프로필, 라우팅 규칙, 작업 로그 | `_system/agents/` |
| 플랫폼 운영 문서 | `_system/platform/` |
| 운영 전략, 로드맵, 타임라인 | `01_ops/strategy/` |
| 초기 아이디어, 과거 기획 흔적 | `01_ops/strategy/history/` |
| 운영 회의록 | `01_ops/meetings/` |
| 모집 공고, 안내 문안, 게시용 copy | `04_studio/posts/` |
| 민감 메모, founder notes | `01_ops/private/` |
| 재사용 문서 템플릿 | `01_ops/templates/` |
| 아직 번호가 없는 세션 아이디어 | `02_sessions/backlog/` |
| 진행 중인 정기 세션 산출물 | `02_sessions/{슬러그}/` |
| 일회성/비정기 세션 | `02_sessions/adhoc/` |
| 프로젝트 문서와 로그 | `03_projects/PRJ-{이름}/` |
| 브랜드 가이드, 네이밍, 디자인 노트 | `04_studio/brand/` |
| 이미지, 로고, concept asset | `04_studio/assets/` |
| 웹/시각 템플릿 | `04_studio/templates/` |
| 최종 export 산출물 | `04_studio/exports/` |
| 게시용 문안, 공지 초안, 배포 copy | `04_studio/posts/` |
| 방법론 리서치 | `05_library/research/methods/` |
| 툴 비교/리뷰 | `05_library/research/tools/` |
| 외부 벤치마크, 참고 자료 | `05_library/research/references/` |
| 발행 완료본 | `05_library/published/` |
| 시즌 백업, legacy hold | `05_library/seasons/` |
| 분류 보류 | `06_inbox/triage/` |
| 외부에서 막 들어온 캡처 | `06_inbox/captures/` |

## 3. 키워드 해석

| 키워드 | 현재 해석 |
|--------|-----------|
| `strategy`, `roadmap`, `timeline` | `01_ops/strategy/` |
| `meeting`, `kickoff`, `review` | `01_ops/meetings/` 또는 `01_ops/reviews/` |
| `recruit`, `announcement`, `copy` | `01_ops/recruit/` 또는 `04_studio/posts/` |
| `session`, `plan`, `record`, `materials` | `02_sessions/` |
| `project`, `migration`, `setup` | `03_projects/` |
| `brand`, `naming`, `logo`, `guide` | `04_studio/brand/` |
| `asset`, `concept`, `export` | `04_studio/assets/` 또는 `04_studio/exports/` |
| `methods`, `study`, `map`, `reference`, `tool` | `05_library/research/` |
| `archive`, `legacy`, `snapshot`, `backup` | `05_library/seasons/` |

## 4. 애매할 때 우선순위

1. 규칙/시스템 문서면 `_system/`
2. 현재 운영 실행 문서면 `01_ops/` 또는 `02_sessions/`
3. 시각 자산이면 `04_studio/`
4. 참고/보존 성격이면 `05_library/`
5. 판단 불가면 `06_inbox/triage/`
