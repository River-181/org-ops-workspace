# 세션 준비 워크플로우

> S00-OT(2026-03-22) 준비 사례를 기반으로 확립된 세션 준비 전체 흐름.
> 이후 모든 세션 준비의 기준 프로토콜로 사용한다.

---

## 전체 흐름

```
[D-21~] 기획 단계
  ↓ session-setup 스킬
  ↓ @prep 에이전트

[D-14~] 콘텐츠 제작 단계
  ↓ 슬라이드 MD 작성
  ↓ slide-prep 스킬 → 디자인 스펙 + 이미지 가이드 + Gemini 프롬프트
  ↓ 스크립트 작성

[D-10~] 외부 발행 단계
  ↓ 모집 공고 작성 (@publish 에이전트)
  ↓ 구글 폼 제작
  ↓ Discord 발행

[D-7~] 외주/위임 단계
  ↓ 키노트 외주 의뢰 (슬라이드 MD 전달)
  ↓ Gemini 다이어그램 생성 (파일 내 프롬프트 사용)
  ↓ 이미지 수집 (Unsplash, 스크린샷)

[D-3] 점검 단계
  ↓ session-launch 스킬

[D-1] 최종 준비
  ↓ 키노트 최종 확인, PDF 예비 저장
  ↓ FigJam 보드 테스트
  ↓ Discord 공지

[D-0] 세션 당일

[D+1~] 기록 단계
  ↓ session-record 스킬
  ↓ 에이전트 메모리 업데이트
```

---

## 단계별 상세

### 1. 기획 단계 (D-21~)

**산출물**: `plan.md`

**주요 결정 사항:**
- 세션 번호, 제목, 일시, 장소 확정
- 진행자 역할 분담 (주용 / 동하 구조 참고)
- 세션 성격 확정 (설명 vs 쌍방향 vs 워크숍)
- 아젠다 및 시간 배분

**실행:**
```
session-setup [세션번호] [제목]
```

---

### 2. 콘텐츠 제작 단계 (D-14~)

**산출물**: `materials/슬라이드.md`, `materials/스크립트.md`

#### 슬라이드 MD 작성 원칙

- **구조 먼저**: 파트 분리 + 장 수 확정 후 콘텐츠 채우기
- **다이어그램 기준**: 관계·구조·흐름이 복잡할 때만. 단순 메시지는 텍스트
- **범례 통일**:
  - `[IMG: 설명]` — 권장 이미지
  - `[SCREENSHOT: 설명]` — 툴 화면 캡처
  - `[DIAGRAM]` — 다이어그램
  - `[ICON: 설명]` — 아이콘

#### slide-prep 스킬 실행 시점

슬라이드 콘텐츠가 80% 이상 완성된 후 실행:

```
slide-prep [파일 경로]
```

결과: 디자인 스펙 섹션 + 이미지 가이드/Gemini 프롬프트 콜아웃 삽입됨

#### 스크립트 원칙 (S00-OT에서 확립)

- **대본이 아닌 흐름 지도** — 문장 단위 대본 금지, 키워드·흐름 수준
- `{...}` 표시 = 상황 판단 포인트 (제거하지 않기)
- 진행자별 담당 구분 명시

---

### 3. 외부 발행 단계 (D-10~)

**산출물**: 모집 공고, 구글 폼, Discord 게시

**순서:**
1. 구글 폼 제작 (자기소개 / 사전 설문)
2. 모집 공고 MD 작성 (`04_studio/posts/YYMMDD-모집공고.md`)
3. Discord 발행 → 실제 링크를 슬라이드 MD에 반영
4. 발행 후 `active-context.md` 갱신

**S00-OT 교훈**: 폼 링크·Discord 링크는 발행 완료 후 슬라이드 파일에 직접 삽입해야 한다.
`[링크]` 플레이스홀더가 남아 있으면 외주 전달 전에 반드시 교체.

---

### 4. 외주/위임 단계 (D-7~)

**키노트 외주 제작**

전달 방법 선택 (우선순위 순):
1. **NotebookLM 슬라이드 만들기** — MD 파일 업로드 → AI 슬라이드 생성 → 키노트 정제
2. **Figma Make AI** — MD 파일 + 디자인 스펙 입력 → 시각 디자인 생성
3. 인력 외주 — 슬라이드 MD + 디자인 스펙 전달

**파트별 분리 제작** 권장 (한 번에 전체보다 파트별이 더 정밀함)

**Gemini 다이어그램 생성**

슬라이드 파일 내 `[!example]- 🤖 Gemini 다이어그램 프롬프트` 콜아웃의 프롬프트를 복사하여
Gemini에 입력 → 이미지 저장 → `04_studio/assets/sessions/S{NN}/` 에 보관

**이미지 수집**

슬라이드 파일 내 `[!tip] 📸 이미지 가이드` 콜아웃 참조:
- Unsplash: 검색어 사용
- 스크린샷: 직접 촬영
- 수집 후 동일 경로(`04_studio/assets/sessions/S{NN}/`)에 보관

---

### 5. 점검 단계 (D-3)

```
session-launch [세션 폴더 경로] 3
```

---

### 6. 최종 준비 (D-1)

- [ ] 키노트 최종 PDF 저장 (예비본 — 프로젝터 호환성 문제 대비)
- [ ] FigJam 보드 / 보조 협업 툴 연결 테스트
- [ ] Discord 당일 공지 발행
- [ ] 진행자 간 최종 역할 확인 (주용·동하 구분 재확인)

---

### 7. 세션 후 기록 (D+1)

```
session-record [세션 폴더 경로]
```

기록 항목:
- 실제 참석자 수, 반응
- 핵심 발언·토론 요약
- 활동 아이디어 클러스터링 결과
- 다음 세션으로 이어지는 액션 아이템
- `change-log.md` append

---

## 역할 분담 패턴 (S00-OT 확립)

[클럽명] 세션은 **2인 진행** 구조를 기본으로 한다:

| 역할 | 담당 스타일 | 주요 파트 |
|------|-----------|---------|
| 큰 그림 진행자 | 철학·전체 맥락 전달 | 클럽 소개, 활동 아이디어 수집 |
| 딸깍 진행자 | 욕망·실용 예시 | 아이스브레이킹, 툴 시연 |

**원칙**: 설명회가 아닌 쌍방향 수집. 참가자 발언 시간이 진행자 발언 시간보다 많아야 함.
