# 세션 준비 워크플로우

> 세션 준비 전체 흐름 프로토콜.
> 이후 모든 세션 준비의 기준 프로토콜로 사용한다.

---

## 전체 흐름

```
[D-21~] 기획 단계
  ↓ session-setup 스킬
  ↓ @prep 에이전트

[D-14~] 콘텐츠 제작 단계
  ↓ 슬라이드 MD 작성
  ↓ slide-prep 스킬 → 디자인 스펙 + 이미지 가이드 + 다이어그램 프롬프트
  ↓ 스크립트 작성

[D-10~] 외부 발행 단계
  ↓ 모집 공고 작성 (@publish 에이전트)
  ↓ 참가 폼 제작
  ↓ 플랫폼 발행

[D-7~] 외주/위임 단계
  ↓ 키노트 외주 의뢰 (슬라이드 MD 전달)
  ↓ 다이어그램 생성 (파일 내 프롬프트 사용)
  ↓ 이미지 수집

[D-3] 점검 단계
  ↓ session-launch 스킬

[D-1] 최종 준비
  ↓ 발표 자료 최종 확인, PDF 예비 저장
  ↓ 보조 툴 테스트
  ↓ 당일 공지

[D-0] 세션 당일

[D+1~] 기록 단계
  ↓ session-record 스킬
  ↓ 에이전트 메모리 업데이트
```

---

## 단계별 상세

### 1. 기획 단계 (D-21~)

**산출물**: `plan.md`

**주요 결정 사항:**
- 세션 번호/식별자, 제목, 일시, 장소 확정
- 진행자 역할 분담
- 세션 성격 확정 (설명 vs 쌍방향 vs 워크숍)
- 아젠다 및 시간 배분

**실행:**
```
session-setup [세션번호] [제목]
```

---

### 2. 콘텐츠 제작 단계 (D-14~)

**산출물**: `materials/슬라이드.md`, `materials/스크립트.md`

#### 슬라이드 MD 작성 원칙

- **구조 먼저**: 파트 분리 + 장 수 확정 후 콘텐츠 채우기
- **다이어그램 기준**: 관계·구조·흐름이 복잡할 때만. 단순 메시지는 텍스트
- **범례 통일**:
  - `[IMG: 설명]` — 권장 이미지
  - `[SCREENSHOT: 설명]` — 툴 화면 캡처
  - `[DIAGRAM]` — 다이어그램
  - `[ICON: 설명]` — 아이콘

#### slide-prep 스킬 실행 시점

슬라이드 콘텐츠가 80% 이상 완성된 후 실행:

```
slide-prep [파일 경로]
```

---

### 3. 외부 발행 단계 (D-10~)

**산출물**: 모집 공고, 참가 폼, 플랫폼 게시

**순서:**
1. 참가 폼 제작
2. 모집 공고 MD 작성 (`01_ops/recruit/YYMMDD-모집공고.md`)
3. 플랫폼 발행 → 실제 링크를 슬라이드 MD에 반영
4. 발행 후 `active-context.md` 갱신

**주의**: 폼 링크·플랫폼 링크는 발행 완료 후 슬라이드 파일에 직접 삽입해야 한다.
`[링크]` 플레이스홀더가 남아 있으면 외주 전달 전에 반드시 교체.

---

### 4. 외주/위임 단계 (D-7~)

**발표 자료 외주 제작**

전달 방법 선택 (우선순위 순):
1. **NotebookLM 슬라이드 만들기** — MD 파일 업로드 → AI 슬라이드 생성
2. **Figma Make AI** — MD 파일 + 디자인 스펙 입력 → 시각 디자인 생성
3. 인력 외주 — 슬라이드 MD + 디자인 스펙 전달

**파트별 분리 제작** 권장 (한 번에 전체보다 파트별이 더 정밀함)

---

### 5. 점검 단계 (D-3)

```
session-launch [세션 폴더 경로] 3
```

---

### 6. 최종 준비 (D-1)

- [ ] 발표 자료 최종 PDF 저장 (예비본)
- [ ] 보조 협업 툴 연결 테스트
- [ ] 당일 공지 발행
- [ ] 진행자 간 최종 역할 확인

---

### 7. 세션 후 기록 (D+1)

```
session-record [세션 폴더 경로]
```

기록 항목:
- 실제 참석자 수, 반응
- 핵심 발언·토론 요약
- 다음 세션으로 이어지는 액션 아이템
- `change-log.md` append
