# Workspace OS 업그레이드 프로토콜

## 1. 버전 관리

- `CLAUDE.md` 상단에 `Workspace OS vX.Y` 버전 명시.
- 구조 변경 시 minor 버전 범프 (v2.1 → v2.2).
- 폴더 체계 변경 시 major 버전 범프 (v2.x → v3.0).

## 2. 구조 변경 절차

1. **결정 기록**: `_system/agents/memory/decisions/YYMMDD-결정제목.md` 작성
2. **인덱스 갱신**: `_system/agents/indexes/` 관련 인덱스 업데이트
3. **CLAUDE.md 갱신**: 버전 범프 + 변경 반영
4. **에이전트/스킬 갱신**: `.claude/agents/` 또는 `.claude/skills/` 수정
5. **Change Log**: `_system/agents/memory/change-log.md`에 날짜 섹션 append
6. **Git 커밋**: 변경사항 커밋 + 필요 시 태그

## 3. 건강 점검 자동화

`/vault-health` 스킬이 다음을 감지한다:
- 절대경로 (`/Users/` 등) 잔존
- 구 네이밍 (`OPS-DOC-`, `_live`, `_arch`) 잔존
- 5레벨 초과 깊이 위반
- frontmatter 미비
- 깨진 위키링크

## 4. 시즌 전환

1. 현재 시즌 문서를 `05_library/seasons/YYYY-SN/`에 스냅샷
2. 해당 문서 frontmatter `status: archived`로 변경
3. `_system/agents/memory/active-context.md` 초기화 (새 시즌용)
4. CLAUDE.md 버전 범프

## 5. 이식성 원칙

- `.claude/` 및 `_system/` 파일에 절대경로 금지. 모든 경로는 볼트 루트 기준 상대경로.
- 머신별 설정(.obsidian/, .DS_Store)은 .gitignore로 제외.
- 전역 Claude 설정(`~/.claude/`)에 의존하지 않는다.
