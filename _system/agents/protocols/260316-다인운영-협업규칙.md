# 다인 운영 협업 규칙

## 개요

운영팀 2-4명이 동일 볼트를 각자 로컬에서 사용하는 상황을 위한 규칙.

## 볼트 공유 방식

### Git 기반 (권장)

```
GitHub (origin) ← push/pull → 운영자 A (로컬)
                ← push/pull → 운영자 B (로컬)
```

- `main` 브랜치가 SoT.
- 각자 작업 후 commit + push.
- 충돌 시 pull → merge → push.

### 클라우드 동기화 (대안)

iCloud/Google Drive/Syncthing로 폴더 동기화.
- 장점: 실시간 반영
- 단점: 충돌 관리가 수동
- `.obsidian/`은 동기화에서 제외 (머신별 설정)

## 에이전트 호출 규칙

- 한 시점에 **한 명만** 에이전트를 호출한다 (메모리 충돌 방지).
- 에이전트 작업 완료 후 바로 commit + push.
- 다른 운영자는 pull 후 작업 시작.

## 파일 소유 관행

| 영역 | 주 담당자 | 보조 |
|------|---------|------|
| 전략/기획 | 대표 | 전원 리뷰 |
| 세션 기획 | 해당 진행자 | 대표 리뷰 |
| 회의록 | 작성 담당자 | — |
| 리서치 | 자유 (태그로 구분) | — |
| 브랜드/디자인 | 디자인 담당 | 대표 리뷰 |

## 충돌 해결

1. **같은 파일 동시 수정**: Git merge로 해결. 해결 불가 시 양쪽 보존 후 수동 병합.
2. **구조 변경 의견 충돌**: `_system/agents/memory/decisions/`에 결정 기록. 대표가 최종 결정.
3. **에이전트 메모리 충돌**: active-context는 항상 최신 파일이 우선.

## 온보딩 (새 운영자)

1. GitHub에서 repo clone
2. Obsidian으로 볼트 열기
3. `CLAUDE.md` 읽기 → `_system/obsidian/mocs/MOC-홈.md`부터 탐색
4. `.claude/` 설정 자동 적용
5. 첫 작업은 `01_ops/members/`에 자기 프로필 추가
