# 프론트매터 계약 v2

> 이 볼트 모든 `.md` 파일의 frontmatter 규칙.
> 에이전트와 스킬은 이 계약을 기준으로 진단·수정한다.

---

## 필수 필드 (모든 파일)

```yaml
---
title: 문서 제목
status: draft | live | archived
created: YYYY-MM-DD
---
```

| 필드 | 값 | 설명 |
|---|---|---|
| `title` | 문자열 | 파일명과 동일하지 않아도 됨 |
| `status` | `draft` / `live` / `archived` | 파일명 접미사 대신 이 필드로 상태 관리 |
| `created` | `YYYY-MM-DD` | 최초 생성일 |

## 권장 필드 (공통)

```yaml
updated: YYYY-MM-DD
kind: meeting | session | project | research | inbox | moc | map | log | system
area: ops | sessions | projects | library | studio | inbox | system
up: "[[부모문서]]"
tags:
  - namespace/tag
related:
  - "[[관련문서1]]"
  - "[[관련문서2]]"
aliases:
  - 검색 별칭
```

### 필드 설명

#### `kind` — 문서 유형

| 값 | 의미 |
|---|---|
| `meeting` | 회의록 |
| `session` | 세션 기획·기록·자료 |
| `project` | 프로젝트 문서 |
| `research` | 리서치·연구 노트 |
| `inbox` | 분류 전 임시 파일 |
| `moc` | MOC — 도메인 허브 |
| `map` | map/index 노드 — 중간 주제 클러스터 |
| `log` | 로그·이력 파일 |
| `system` | 볼트 시스템·설정 문서 |

#### `area` — 폴더 분류

| 값 | 폴더 |
|---|---|
| `ops` | `01_ops/` |
| `sessions` | `02_sessions/` |
| `projects` | `03_projects/` |
| `library` | `05_library/` |
| `studio` | `04_studio/` |
| `inbox` | `06_inbox/` |
| `system` | `_system/` |

#### `up` — 부모 문서 (단수)

- **단수 문자열** — 부모는 1개만.
- 계층: `문서 → map/index → MOC → MOC-홈`

```yaml
# 올바른 예
up: "[[MOC-운영]]"

# 금지 — 복수 사용 불가
up:
  - "[[MOC-운영]]"
  - "[[MOC-프로젝트]]"
```

#### `related` — 관련 문서

- `up:`으로 표현할 수 없는 수평적 관계.
- 포함 기준:
  - 같은 주제의 다른 관점
  - 선행·후속 관계 (예: 기획안 → 기록)
  - 같은 세션의 다른 파일

#### `tags` — 태그

- 반드시 `tag-registry.md`에 등록된 태그만 사용.
- 네임스페이스 필수: `namespace/tag` 형식.
