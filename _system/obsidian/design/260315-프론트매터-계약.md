# 프론트매터 계약 v2

> 이 볼트 모든 `.md` 파일의 frontmatter 규칙.
> 에이전트와 스킬은 이 계약을 기준으로 진단·수정한다.

---

## 필수 필드 (모든 파일)

```yaml
---
title: 문서 제목
status: draft | live | archived
created: YYYY-MM-DD
---
```

| 필드 | 값 | 설명 |
|---|---|---|
| `title` | 문자열 | 파일명과 동일하지 않아도 됨 |
| `status` | `draft` / `live` / `archived` | 파일명 접미사 대신 이 필드로 상태 관리 |
| `created` | `YYYY-MM-DD` | 최초 생성일 |

## 권장 필드 (공통)

```yaml
updated: YYYY-MM-DD
kind: meeting | session | project | research | inbox | moc | map | audit | log | system | drop | tool | guide
area: ops | sessions | projects | library | studio | inbox | system
up: "[[부모문서]]"
tags:
  - namespace/tag
related:
  - "[[관련문서1]]"
  - "[[관련문서2]]"
aliases:
  - 검색 별칭
```

### 필드 설명

#### `kind` — 문서 유형

| 값 | 의미 |
|---|---|
| `meeting` | 회의록 |
| `session` | 세션 기획·기록·자료 |
| `project` | 프로젝트 문서 |
| `research` | 리서치·연구 노트 |
| `inbox` | 분류 전 임시 파일 |
| `moc` | **MOC** — 도메인 전체 큐레이션 허브. 수동 링크. `_system/obsidian/mocs/` |
| `map` | **MAP** — Base 중심 동적 탐색 허브. `_system/obsidian/maps/` |
| `audit` | **Audit** — Dataview 기반 감사·누락·편중 점검 노트. `_system/obsidian/audits/` |
| `index` | **INDEX** — 속성 기반 자동 목록. Dataview 전용. 해당 데이터 폴더 내부 |
| `log` | 로그·이력 파일 |
| `system` | 볼트 시스템·설정 문서 |
| `drop` | Drop — 큐레이션 공유 발행물 (`04_studio/drops/`) |
| `article` | Article — 내부 생산 지식 결과물. 블로그형, 대외 공개 가능 (`05_library/articles/`) |
| `reference` | Reference — 외부 소스 자료 보관 (`05_library/research/references/`) |
| `post` | Post — 운영 공지·발행 콘텐츠 (`01_ops/posts/`) |
| `member` | Member — 멤버 프로필 (`01_ops/members/`) |
| `tool` | Tool — 도구 인덱스/허브 (`_system/tools/*/*.md`) |
| `guide` | Guide — 상세 운영 문서 |

#### `area` — 폴더 분류

폴더 구조와 1:1 대응. 사용자 편의 이름이 아닌 **시스템 식별자**.

| 값 | 폴더 |
|---|---|
| `ops` | `01_ops/` |
| `sessions` | `02_sessions/` |
| `projects` | `03_projects/` |
| `library` | `05_library/` |
| `studio` | `04_studio/` |
| `inbox` | `06_inbox/` |
| `system` | `_system/` |

#### `up` — 부모 문서 (단수)

- **단수 문자열** — 부모는 1개만.
- 계층: `문서 → 단위 허브(map/index) → MOC → MOC-홈`
- 부모가 없는 최상위 문서(MOC-홈 등)는 생략 가능.

```yaml
# 올바른 예
up: "[[MOC-운영]]"

# 금지 — 복수 사용 불가
up:
  - "[[MOC-운영]]"
  - "[[MOC-프로젝트]]"
```

부모가 애매할 경우: 더 직접적인 주제 문서를 선택. 나머지 관계는 `related:`로.

#### `related` — 관련 문서

- `up:`으로 표현할 수 없는 수평적 관계.
- 포함 기준:
  - 같은 주제의 다른 관점
  - 선행·후속 관계 (예: 기획안 → 기록)
  - 같은 세션의 다른 파일
  - 참고·인용 자료

```yaml
related:
  - "[[260318-세션1기획]]"
  - "[[260318-세션1자료]]"
```

#### `aliases` — Obsidian 검색 별칭

- Obsidian의 빌트인 기능. `[[wikilink]]` 검색 시 이 이름으로도 찾힌다.
- **`area:`와 완전히 다른 목적** — `area:`는 시스템 분류, `aliases:`는 검색 편의.
- 필요할 때만 사용 (한국어 약칭, 영문 표기 등).

```yaml
aliases:
  - 프매계약
  - frontmatter-contract
```

#### `tags` — 태그

- 반드시 `tag-registry.md`에 등록된 태그만 사용.
- 네임스페이스 필수: `namespace/tag` 형식.
- 본문에서 `#namespace/tag` 형식으로도 사용 가능 (Obsidian Tags 패널 집계).

#### `session_status` — 세션 칸반 상태

세션의 칸반 진행상태. 칸반-세션 보드의 컬럼과 1:1 매핑.

| 값 | 의미 | 칸반 컬럼 |
|---|---|---|
| `idea` | 백로그 아이디어 | 💡 아이디어 |
| `planning` | 기획 진행 중 | 📋 기획 중 |
| `prep` | 자료 준비 중 | 🏃 준비 중 |
| `done` | 세션 완료 | ✅ 완료 |

대상 파일: `02_sessions/S*/{세션명}-plan.md`, `02_sessions/backlog/*.md`, `02_sessions/adhoc/*-plan.md`

| 값 | 의미 | 칸반 컬럼 |
|---|---|---|

| `active` | 진행 중 | 🔄 진행 중 |
| `review` | 검토 중 | 👁 검토 |
| `done` | 완료 | ✅ 완료 |
| `paused` | 보류 | ⏸ 보류 |

대상 파일: `03_projects/PRJ-*/README.md`

### 도구 (`_system/tools/`)

```yaml
kind: tool
area: system
up: "[[MAP-도구]]"
tool_id: nlm | figma | kanban | github
tool_type: cli | mcp | script
owner: ops | research | studio
auth_mode: none | session | oauth | gh-auth
verified_on: YYYY-MM-DD
image: "[[_system/tools/assets/tool-*.svg]]"
icon: 문자 또는 짧은 심볼
env_keys:
  - ENV_NAME
```

- `tool` 노트는 도구 자체의 인덱스이자 허브다.
- 상세 운영가이드는 `kind: guide`로 두고, `up:`은 해당 `tool` 노트를 가리킨다.
- `image`는 `tools.base` 카드 뷰용 대표 이미지다. 가능하면 로컬 자산을 사용한다.
- `icon`은 카드 뷰 보조 식별자다. 이미지가 보이지 않을 때도 도구를 빠르게 구분할 수 있게 한다.

### MOC / MAP / INDEX 노드

세 개념은 **진입점 계층**을 구성한다. 루만의 제텔카스텐 인덱스처럼, 폴더 브라우저 없이도 이 계층으로 볼트 전체에 진입할 수 있어야 한다.

| 개념 | 역할 | 관리 | 위치 |
|------|------|------|------|
| **MOC** | 도메인 허브 — "이 영역에서 무엇을 볼까" | 수동 큐레이션 | `_system/obsidian/mocs/` |
| **MAP** | 주제 클러스터 허브 — 관련 파일 3개↑ 묶음 | Dataview + 수동 보완 | `_system/obsidian/maps/` |
| **INDEX** | 속성 기반 자동 목록 | Dataview 전용 | 해당 데이터 폴더 내부 |

계층: `개별 문서 → 단위 허브 → MAP/MOC → MOC-홈`

**단위 허브 파일** = 하나의 프로젝트·세션·주제를 묶는 파일. 폴더명과 동일한 이름 사용.
상세 규칙: [[260327-폴더진입점-대원칙]]

```yaml
# MOC
kind: moc
area: (도메인)
up: "[[MOC-홈]]"

# MAP
kind: map
area: system
up: "[[MOC-시스템]]"   # 상위 MOC를 가리킴

# INDEX
kind: index
area: (해당 폴더 area)
up: "[[상위 MAP 또는 MOC]]"
```
