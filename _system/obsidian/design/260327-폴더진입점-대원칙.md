# 폴더 진입점 파일 대원칙

> 볼트 내 모든 "단위를 묶는 파일"의 이름, 역할, 계층에 관한 대원칙.
> 이 문서가 README.md와 Obsidian 진입점 파일의 혼동을 해소한다.

---

## 핵심 원칙: 두 종류의 대표 파일

볼트에서 "폴더를 대표하는 파일"은 **목적이 다른 두 종류**로 명확히 분리한다.

| 종류 | 이름 | 독자 | 역할 |
|------|------|------|------|
| **README.md** | 항상 `README.md` | git, Claude Code | 폴더 구조 설명, 에이전트 지침 |
| **진입점 파일** | 폴더명 또는 단위 고유명 | Obsidian, 사람 | 하위 파일 허브, 그래프 노드 |

**README.md는 Obsidian 노드가 아니다.** `up:`, `kind:` frontmatter 불필요.
모든 Base 필터에 `file.name != "README"` 적용.

---

## 3층 계층 구조

```
콘텐츠 파일 (글, 기록, 분석, 자료)
    ↓ up:
단위 허브 파일  ←── 이 대원칙의 핵심
    ↓ up:
MOC (도메인 전체 허브)
    ↓ up:
MOC-홈
```

---

## 단위 허브 파일 규칙

### 이름 규칙: **폴더명 = 파일명**

하나의 단위를 대표하는 파일은 **그 폴더와 같은 이름**을 갖는다.

| 단위 유형 | 폴더 예시 | 진입점 파일 | kind |
|----------|----------|-----------|------|
| 프로젝트 | `03_projects/PRJ-볼트v3/` | `PRJ-볼트v3.md` | `project` |
| 세션 | `02_sessions/S01-내우주의시작/` | `S01-내우주의시작.md` | `session` |
| 리서치 주제 | `05_library/research/ai/` | `ai.md` | `index` |
| 리서치 주제 | `05_library/research/methods/일정관리/` | `일정관리.md` | `index` |
| 멤버 공간 | `05_library/members/` | `members.md` | `index` |
| 아티클 공간 | `05_library/articles/` | `articles.md` | `map` |

### up 연결 규칙

단위 허브 파일의 `up:`은 해당 도메인 MOC를 직접 가리킨다.

```yaml
# 프로젝트 허브 파일 예시
up: "[[MOC-프로젝트]]"

# 리서치 주제 허브 파일 예시
up: "[[MOC-라이브러리]]"

# 세션 허브 파일 예시
up: "[[MOC-세션]]"
```

하위 콘텐츠 파일은 허브 파일을 가리킨다:
```yaml
# 프로젝트 내 문서
up: "[[PRJ-볼트v3]]"

# 세션 내 기획안
up: "[[S01-내우주의시작]]"
```

## 예외 및 특수 케이스

### 섹션 레벨 폴더 (01_ops/, 05_library/ 등)

최상위 섹션 폴더 (`01_ops/`, `02_sessions/` 등)는 MOC가 대신 허브 역할을 한다.
이 레벨에 별도 진입점 파일을 만들지 않는다.

```
01_ops/                 ← 진입점 파일 없음 (README.md만)
    └── README.md       ← git용 폴더 설명 only
MOC-운영.md             ← 01_ops/ 전체를 커버하는 허브
```

### 세션 기획안·기록 (plan.md, record.md)

세션 폴더 내 `plan.md`, `record.md`는 **콘텐츠 파일**이다.
이들의 `up:`은 해당 세션의 단위 허브를 가리켜야 한다.

```
02_sessions/S04-제목/
├── S04-제목.md         ← 세션 단위 허브 (up: MOC-세션)
├── plan.md             ← 콘텐츠 (up: S04-제목)
├── record.md           ← 콘텐츠 (up: S04-제목)
└── materials/
    └── ...             ← 콘텐츠 (up: S04-제목 or plan)
```

### 기존 세션 (S01~S03)

기존 세션 폴더에는 단위 허브 파일이 없다. Phase 2 작업에서 생성한다.
임시로 `plan.md`의 `up:`이 `MOC-세션`을 가리켜도 무방.

## 현재 상태 및 적용 로드맵

### 이미 올바른 것 ✅

| 파일 | 이유 |
|------|------|
| `03_projects/PRJ-*/PRJ-이름.md` | 폴더명 = 파일명, `up: MOC-프로젝트` |
| `_system/obsidian/mocs/MOC-*.md` | `kind: moc`, `up: MOC-홈` |

### 즉시 수정 필요 (Phase 2) ⚠️

| 현재 파일 | 변경 후 | 이유 |
|----------|---------|------|
| `05_library/research/ai/README.md` | `ai.md` | Obsidian 그래프 노드, 이름이 README |
| `05_library/research/methods/일정관리/README.md` | `일정관리.md` | 동일 |
| `05_library/research/methods/파일링-시스템/README.md` | `파일링-시스템.md` | 동일 |
| `05_library/members/README.md` | `members.md` | `kind: index` 있음, 이름만 README |
| `05_library/articles/README.md` | `articles.md` | `kind: map` 있음, 이름만 README |
| `02_sessions/S01~S03/` | `S0N-제목.md` 신규 생성 | 세션 단위 허브 없음 |

### 변경하지 않는 README.md 🔵

섹션 레벨 README.md (`01_ops/`, `02_sessions/`, `05_library/`, `_system/` 등)는
git·Claude Code 설명 전용으로 유지. Obsidian에서 보이지 않아도 됨.

## Base 필터 규칙

모든 `.base` 파일에서 README.md는 제외한다:

```yaml
filters:
  and:
    - file.inFolder("폴더경로")
    - file.name != "README"
```

진입점 파일을 Base에서 올바르게 잡으려면 `up:` 필드 필터가 가장 정확하다:

```yaml
filters:
  and:
    - up == link("MOC-프로젝트")    # 프로젝트 허브만
```
