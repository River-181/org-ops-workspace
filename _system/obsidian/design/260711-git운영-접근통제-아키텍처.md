# [볼트명] Git 운영·접근통제 아키텍처

> git·GitHub 접근통제 설계 기록. 커밋·푸시 실행 권한 및 미결 리스크의 정본.

---

## 1. 핵심 원칙

- git이 이 워크스페이스의 **단일 진실원천(SoT)**이다.
- **쓰기(commit/push) 권한은 [운영자]의 맥 단독.**
- 에이전트(AI 사령탑)와 운영팀은 git을 직접 쓰지 못한다 — 우연이 아니라 설계다.

## 2. Push 흐름

```
에이전트/운영팀 (VM·각 머신)     [운영자] 맥                         GitHub
─────────────────────────      ───────────────────────        ──────────
Vault([볼트명])에 파일 쓰기 ─▶ Syncthing이 미러링         ─▶  (여기까지 자동)
                                [볼트-경로]
                                (이곳에만 .git + gh 토큰)
                                └─ git commit + push       ─────▶ origin/main
                                   ↑ 이 스텝만 사람/Claude가 맥에서 실행
```

- 에이전트는 **절대 push하지 않는다.** VM에는 `.git`도, `gh`도, SSH 키도 없다 — 일부러 안 둔다.

## 3. GitHub 접근통제

- repo: **Private**
- 단독 admin/writer: **[운영자계정]**
- collaborator **[협업자계정]**: write 권한이나, main이 PR 게이트라 **직접 push 불가 → PR 제출 → [운영자] 승인**으로만 반영.
- **main 브랜치 보호**: PR 필수(승인 1) · force-push 차단 · 브랜치 삭제 차단 · `enforce_admins=false`([운영자]는 직접 push 우회 가능). 결과: [운영자]=승인 게이트, 그 외=PR만.

## 4. 보안 근거

- 공유 VM에 git 쓰기 자격증명을 두지 않음 → 운영팀·오염된 에이전트에 의한 force-push·브랜치 삭제·토큰 유출 차단.
- `.stignore`(Syncthing 미러 제외): `.git`, PII(`01_ops/members`, `01_ops/finance`, `06_inbox`, `**/*transcript*`), 로컬도구(skillspector·im-not-ai·content-skills·memory index), **mac 잔재 `(?d)` 규칙**(원격 디렉토리 삭제가 로컬 `.DS_Store`/`Icon` 때문에 sync를 막던 문제 해소).
- `.gitignore`(git 제외): 런타임/로컬(`.hermes/`, `.codex/`, `.stversions/`, `.stfolder/`), 시크릿(`_system/tools/.env`, `auth.json` 등), mac 정크(`.DS_Store`, `Icon`).

## 5. 알려진 잔여 리스크

> ⚠️ PII가 **git 히스토리에 이미 커밋돼 있을 수 있다**: `01_ops/members`, `06_inbox`, transcript 파일 등.
> `.stignore`는 Syncthing 미러만 막을 뿐 git 히스토리는 막지 못한다 → collaborator가 clone하면 열람 가능하다.
> 필요 시 `git filter-repo`로 히스토리 스크럽 검토. **[운영자] 판단 필요.**

## 6. 커밋/푸시 정책

- 단일 writer: [운영자] 맥 단독. 에이전트는 콘텐츠·change-log 초안만 만든다.
- main: [운영자] 직접 push(admin 우회) / 그 외는 PR + [운영자] 승인.
- 주기: Syncthing 누적분을 세션마감·주간 단위로 일괄 커밋(파일 단위 커밋 X).
- 절대 커밋 금지: 런타임(`.hermes/`·`.codex/`·`.stversions/`·`.stfolder/`), 시크릿(`.env`·토큰·키), PII(§5 재검토 전까지).
- 커밋 전 안전점검: 런타임/정크 staged 여부 확인 + 시크릿 패턴 스캔.
- 커밋 메시지: conventional commits(`feat`/`fix`/`docs`/`chore`).

## 7. 에이전트 운영 수칙 (AI 사령탑용)

- 너는 git/gh/ssh를 실행하지 않는다.
- Vault에 파일을 쓰고 change-log에 변경 요약을 남긴다.
- push는 [운영자] 맥에서 일어난다. **토큰·키를 요청하지 마라.**
