# Claude Code 에이전트·스킬 공식 포맷

> 2026-03-18 조사 기준. Claude Code 공식 문서에서 확인한 내용.

---

## 1. 스킬 (Skills)

### 파일 위치

```
.claude/skills/{skill-name}.md          ← 플랫 파일 (현재 프로젝트 방식)
.claude/skills/{skill-name}/SKILL.md   ← 디렉터리 방식 (공식 최신)
~/.claude/skills/                       ← 유저 전역
```

### Frontmatter (모두 선택)

```yaml
---
description: 이 스킬이 하는 일 (Claude가 자동 호출 여부 판단에 사용)
argument-hint: "[인자 힌트]"             # 자동완성 표시용
disable-model-invocation: true          # true면 Claude 자동 로드 방지
user-invocable: false                   # false면 /메뉴에서 숨김 (Claude 전용)
allowed-tools: Read, Grep, Glob         # 이 스킬 활성 시 자동 허용 툴 목록
model: sonnet                           # sonnet | opus | haiku | 풀 ID | inherit
context: fork                           # fork로 설정 시 서브에이전트 컨텍스트로 실행
agent: Explore                          # context:fork일 때 어느 서브에이전트 타입 사용
hooks:                                  # 스킬 범위 라이프사이클 훅
  PreToolUse: [...]
---
```

### 호출 방법

- `/skill-name` — 직접 호출
- `/skill-name argument` — 인자와 함께 호출
- Claude가 description 보고 자동 호출

### 변수 치환

- `$ARGUMENTS` — 전달된 인자 전체
- `$ARGUMENTS[N]` — N번째 인자
- `$N` — N번째 인자 (단축)
- `${CLAUDE_SESSION_ID}` — 현재 세션 ID
- `${CLAUDE_SKILL_DIR}` — 스킬 파일 위치

### 우선순위

Enterprise > Personal(`~/.claude/`) > Project(`.claude/`) > Plugin

## 2. 에이전트 (Agents / Subagents)

### 파일 위치

```
.claude/agents/{agent-name}.md         ← 플랫 파일
.claude/agents/{agent-name}/AGENT.md  ← 디렉터리 방식
~/.claude/agents/                       ← 유저 전역
```

### Frontmatter

```yaml
---
description: 이 에이전트가 하는 일 (REQUIRED — 위임 판단에 사용)
tools: Read, Glob, Grep, Bash          # 허용 툴 화이트리스트 (생략 시 전체 상속)
disallowedTools: Write, Edit           # 금지 툴 블랙리스트
model: sonnet                          # sonnet | opus | haiku | 풀 ID | inherit
permissionMode: default                # default | acceptEdits | dontAsk | bypassPermissions | plan
maxTurns: 10                           # 최대 에이전트 턴 수
skills:                                # 프리로드할 스킬 (내용이 시스템 프롬프트에 주입됨)
  - api-conventions
mcpServers:                            # 이 에이전트에서 사용 가능한 MCP 서버
  - slack
memory: user                           # 영속 메모리 범위: user | project | local
background: true                       # true면 항상 백그라운드 실행
isolation: worktree                    # worktree로 설정 시 Git worktree에서 격리 실행
hooks:                                 # 이 에이전트 범위 라이프사이클 훅
  PreToolUse: [...]
  PostToolUse: [...]
  Stop: [...]
---

# 시스템 프롬프트 (마크다운 본문)
에이전트의 역할과 행동 지침을 여기에 작성.
```

### 호출 방법

- Claude가 description 보고 자동 위임
- 명시적 요청: "Use the graph-weaver agent"
- 에이전트는 격리된 컨텍스트 윈도우에서 실행됨

### 영속 메모리 위치

| 설정 | 경로 |
|------|------|
| `memory: user` | `~/.claude/agent-memory/{agent-name}/` |
| `memory: project` | `.claude/agent-memory/{agent-name}/` |
| `memory: local` | `.claude/agent-memory-local/{agent-name}/` |

### permissionMode 의미

| 값 | 의미 |
|----|------|
| `default` | 표준 권한 확인 |
| `acceptEdits` | 파일 편집 자동 승인 |
| `dontAsk` | 확인 프롬프트 자동 거부 (허용된 툴은 작동) |
| `bypassPermissions` | 모든 권한 확인 건너뜀 (주의) |
| `plan` | 읽기 전용 탐색 모드 |

## 3. 스킬 vs 에이전트 vs 플러그인

| | 스킬 | 에이전트 | 플러그인 |
|---|------|---------|---------|
| **호출** | `/이름` 또는 자동 | 자동 위임 또는 명시 요청 | `/플러그인:스킬명` |
| **컨텍스트** | 메인 대화에 인라인 | 격리된 별도 컨텍스트 | 패키지 (스킬/에이전트 포함) |
| **상태 유지** | 세션 한정 | 영속 메모리 가능 | 배포 포맷 |
| **파일 수정** | 제한적 (보고 위주) | 승인 후 수정 | 에이전트 정의에 따름 |
| **용도** | 재사용 프롬프트·워크플로우 | 전문 역할 분리 | 팀·커뮤니티 공유 |

---

## 4. 주요 주의사항

- 에이전트는 CLAUDE.md를 상속하지만 Claude Code 전체 시스템 프롬프트는 받지 않음
- 스킬의 `allowed-tools`는 화이트리스트 — 목록 외 툴은 여전히 권한 요청
- 에이전트의 `tools`도 화이트리스트 — 생략 시 전체 상속
- 플러그인 에이전트에서는 보안상 `hooks`, `mcpServers`, `permissionMode` 미지원
- `context: fork` 스킬은 서브에이전트로 실행 — 메인 컨텍스트 보호에 유리
